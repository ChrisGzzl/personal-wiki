# Personal Wiki — 一个不会"烂尾"的知识库

## 为什么大多数知识库都会烂尾？

你肯定经历过这种事：

- 看到好文章，收藏了，再也没打开过
- 记了一堆笔记，乱七八糟，想找的时候找不到
- 用过各种笔记软件，每次都是开头热情高涨，一周后就荒废了

这不是你的问题，是工具的问题。**现有的知识管理工具，"放进去"太费劲，"整理好"更费劲，"用出来"更没谱。** 费劲的事情，人就是坚持不下去。

Personal Wiki 的核心思路就是：**让存入不费劲，让整理自动做，让使用随时可用。**

---

## 像堆肥一样管理知识

你可以把知识管理想象成堆肥：

1. **随手扔进去**（采集） — 厨余垃圾直接倒进堆肥桶，不用分类
2. **自动发酵**（编译） — 微生物帮你分解，你不用动手
3. **挑好的用**（晋升） — 发酵完了，你挑合格的拿去种花
4. **用的时候拿**（查询） — 种花的时候去堆肥桶取，不需要的时候不用管

对应到系统里：

### 第一阶段：采集 — 回复 `#wiki` 就完事了

你跟 AI 聊天的时候，觉得某段对话有价值，回复 `#wiki`，系统自动把上下文存下来。

- 不需要复制粘贴
- 不需要打开任何 App
- 不需要想"这段该存哪个分类"
- 一句话都不用多打

就像往桶里扔东西一样，扔完就走。系统不会漏掉，因为 `#wiki` 是一个硬性标记，不存在"忘了存"的情况。

### 第二阶段：编译 — AI 帮你整理，你不用管

系统定期（或你手动触发）把攒下来的碎片整理成结构化的笔记草稿。

这里的"编译"跟写代码没关系，你可以理解为：**AI 把你东一句西一句的聊天记录，整理成一篇有条理的笔记。**

关键设计：

- 整理出来的东西**不会直接进入知识库**，只是草稿，放在一个"待审核"的暂存区
- 如果你后来审核的时候觉得某篇草稿不好，你写个拒绝理由，系统会记住，下次整理的时候自动避开同类问题
- 如果暂存区没有新素材，系统什么都不做，不白跑

### 第三阶段：晋升 — 你说行才行，你说不行就不行

这是唯一需要你动手的环节。系统把草稿列出来，你过一遍：

- 觉得有用的 → 确认入库
- 觉得没用的 → 拒绝，顺手写个理由

就这么简单。**AI 不替你做决定。** 哪条知识能进知识库，只有你说了算。

而且拒绝的理由会被反馈回去，AI 下次整理就会自动改进。这意味着：**你拒绝得越多，AI 整理得越好。**

### 第四阶段：查询 — 两种方式，想查就查，不查不浪费

知识存了不用等于没存。但传统知识库的查询要么费劲（手动翻），要么不靠谱（AI 自己猜你要什么）。

Personal Wiki 有两种查询方式：

**方式一：`#ask` — 你主动查，必定触发**

聊天时回复 `#ask 我对知识管理的看法是什么？`，AI 会先去知识库里找，找到的标注📚，自己补充的标注💡。

- 你打了就查，没打就不查，不存在 AI 乱翻的情况
- 知识库没相关内容时，AI 会明确告诉你，不会瞎编
- **确定性触发** — 你想查的时候一定能查到

**方式二：AI 自动查 — 它觉得有关就查**

即使你不打 `#ask`，如果聊到跟你之前沉淀过的知识相关的话题，AI 也会主动去知识库找。

- 不需要你记得去查 — AI 自己判断相关性
- 不是每次都查 — 只有判断话题相关时才会查
- **概率性触发** — 大部分时候有用，但不保证每次都触发

两种方式组合起来就是：**你想查的时候必定能查到（#ask），你想不起来的时候 AI 也有概率帮你查到（自动触发）。** 完美互补。

## 四个设计原则 + 两个入库哲学

这不是随便做的，每个设计背后都有一个原则在支撑：

### 1. 存在成本要趋近于零

"存在成本"就是你什么都不做的时候，这个系统还要消耗你多少精力。

- 没攒够新素材 → 不编译，不费算力
- 草稿放了太久没人审核 → 自动清理，不占地方
- 原始素材没被编译 → 90天后归档，再过90天删掉
- 你没打 `#ask`，也没聊到相关话题 → 知识库安安静静待着，零消耗

**一句话：你不管它，它也不会变成负担。**

### 2. 反馈要"闭环"

"闭环"就是：你给的意见，系统有没有真的听进去。

- 采集阶段：你扔进去就完事了，没有反馈，这是"开环" — 但它也不需要反馈，扔进去就行
- 编译阶段：你拒绝一篇草稿，理由会回传给 AI，下次整理会更准，这是"半闭环"
- 晋升阶段：你做最终决定，AI 只执行，这是"全闭环" — 做决定的和干活的不是同一方，互相独立

**一句话：拒绝不是白拒绝，AI 会学习你的偏好。**

### 3. 触发要"确定"

"触发"就是：系统什么时候开始干活。

- `#wiki` 这个标记是硬性的 — 你打了就存，没打就不存，不会遗漏也不会误触发
- `#ask` 这个标记也是硬性的 — 你打了就查，没打就不查
- 没有新素材就不编译 — 不存在"定时跑一次但是什么都没干"的浪费
- AI 自动查知识库是概率性的 — 有用但不保证，所以重要的事情你还是打 `#ask`

**一句话：关键动作你说了算，AI 只在你不方便的时候帮一把。**

### 4. 每层只做本层的事

- 采集层：只管存，不整理
- 编译层：只管整理，不管好坏
- 晋升层：只管好坏，不整理
- 查询层：只管取用，不修改

不会出现 AI 一边整理一边自己做决定放行的情况。**每一步的职责是清清楚楚的。**

### 5. 半衰期 — 堆肥桶也要定期倒

堆肥不是永远堆着不动的。没长出芽的废料要清理掉，不然堆肥桶就变成了垃圾堆。

- 原始素材 90 天没被整理 → 归档（不再处理，但还留着）
- 归档的素材再过 90 天 → 直接删掉
- 整理好的草稿 180 天没人审核 → 自动拒绝

这不是惩罚，是自然代谢。**真正重要的知识不会被遗忘，因为它会被反复使用。** 长期没人问的东西，说明它没那么重要，留着只会占地方。

### 6. 最小可用知识 — 别什么都在入库

这是反"入库强迫症"的。

很多人用知识库的时候有一种心理："万一以后用得上呢？"于是什么都往里塞。结果知识库变成了仓库，找东西比不用知识库还费劲。

Personal Wiki 的入库标准很简单：**下次我问的时候，我想得到一个精准的答案吗？**

- 能被精准调用的、稳定的、边界清楚的知识 → 入库
- 还在探索中的、模糊的、频繁变动的 → 留在堆肥区，AI 用 `#ask` 也能搜到，但不占知识库的位置

**一句话：入库门槛高，但一旦入库就是可信的。**

## 数据怎么流转的

```
你打 #wiki → 存入原始素材区 → AI 整理成草稿 → 你审核
                                        │
                            你拒绝了 ←──┘──→ 你通过了 → 进入知识库
                                │                        │
                                └→ 拒绝理由回传给 AI      └→ 你打 #ask → AI 从知识库查给你
                                    ，下次改进             └→ 聊到相关话题 → AI 自动查知识库
```

**自动清理规则：**
- 原始素材 90 天没被整理 → 归档（不删，但不再整理）
- 归档的素材再过 90 天 → 删除
- 整理好的草稿 180 天没人审核 → 自动拒绝并告知 AI

这些规则保证：**你不管它，它也不会变成一团乱麻。**

## 和"传统知识库"有什么不一样？

大多数知识库（比如 Notion、Obsidian + 插件）的思路是：**存进去 → 需要的时候搜出来。**

这有两个大问题：第一，你存进去的是碎片，搜出来的也是碎片，质量参差不齐；第二，你需要的时候不一定记得去查，AI 自己查又不可靠。

Personal Wiki 的思路是：**存进去是碎片，但入库的一定是整理好的；你想查的时候必定能查到，你想不起来的时候 AI 也有概率帮你查到。**

| | 传统知识库 | Personal Wiki |
|---|---------|--------|
| 怎么存 | 手动复制粘贴，或自动存一堆没整理的 | 聊天时 `#wiki`，零成本 |
| 存完什么样 | 一堆碎片，越来越乱 | AI 自动整理成草稿 |
| 质量谁把关 | 没人把关，好的坏的全混在一起 | 你亲自审核，只有你说行的才能入库 |
| 怎么查 | 要么手动翻，要么靠 AI 猜你想看什么 | `#ask` 确定能查到 + AI 自动查补充 |
| 整理会越做越好吗 | 不会 | 会，你的拒绝理由会帮 AI 改进 |
| 不管它会怎样 | 越来越乱，最终烂尾 | 自动清理，永远不会变成负担 |

## 怎么用

```bash
# 安装
pip install -e /path/to/personal-wiki
wiki init ~/wiki

# 日常使用
wiki capture --text "今天学到一个道理..."   # 随手记一句
wiki compile                               # 让 AI 整理一下
wiki promote                               # 审核草稿，决定入库
wiki query "我对XX的看法"                    # 从知识库里找答案

# 最简单的方式：
# 聊天时回复 #wiki → 采集
# 聊天时回复 #ask  → 查询
# 什么都不打，聊到相关话题 → AI 也有概率自动查知识库
```
