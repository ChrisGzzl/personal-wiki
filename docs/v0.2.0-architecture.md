# wiki-cli v0.2.0 架构文档

## 数据流

```
┌─────────────┐     #wiki      ┌─────────┐
│  OpenClaw    │ ──────────────→│  raw/   │
│  Plugin      │    capture     │         │
└─────────────┘                └────┬────┘
                                    │
┌─────────────┐    compile     ┌────┴────┐     ┌──────────────────┐
│  wiki       │ ──────────────→│compiled/│←────│compile_feedback.md│
│  compile    │  (guard+LLM)   │YYYY/MM/ │     │(reject signals)  │
└─────────────┘                └────┬────┘     └──────────────────┘
                                    │
┌─────────────┐    promote     ┌────┴────┐
│  wiki       │ ──────────────→│  wiki/  │
│  promote    │  (human gate)  │         │
└─────────────┘                └─────────┘
                                    │
┌─────────────┐    query       ┌────┴────┐
│  wiki       │ ←─────────────│  wiki/  │
│  query      │                └─────────┘
└─────────────┘

┌─────────────┐    gc         ┌─────────────┐
│  wiki gc    │ ─────────────→│ raw/archive/ │ → delete
│             │                │ (90+90 day) │
└─────────────┘                └─────────────┘
```

## Harness 触发矩阵

| 模块 | 命令/触发 | 触发确定性 | 反馈闭环 | 存在成本 |
|------|----------|-----------|----------|----------|
| Capture (#wiki) | OpenClaw Plugin | 确定性 | 开环 | 零 |
| Capture (CLI) | `wiki capture` | 确定性 | 开环 | 零 |
| Compile | `wiki compile` + guard | 确定性 | 半闭环 | 低 |
| Promote | `wiki promote` | 条件性 | 全闭环 | 低 |
| GC | `wiki gc` | 确定性 | 开环 | 零（减负） |

## 目录结构

```
~/wiki/
├── raw/                    # 原始素材（Capture 写入，保留源数据）
│   ├── articles/           # URL 抓取的内容
│   ├── notes/              # 手动输入 / #wiki 捕获
│   ├── papers/             # PDF/论文
│   ├── images/             # 图片
│   ├── misc/               # 其他文件
│   └── archive/            # 90天未编译的归档
├── compiled/               # Compile 产出（暂存区）
│   └── 2026/               # 按年月归档
│       └── 04/
├── wiki/                   # 正式知识库（Promote 确认后）
│   ├── concepts/
│   ├── topics/
│   ├── people/
│   ├── meta/
│   ├── journal/
│   └── index.md
├── outputs/                # 问答存档
├── schema.md               # 知识库规则（人类维护，Agent 只读）
├── config.yaml             # 工具配置
├── compile_feedback.md     # Compile 反馈信号（拒绝原因回灌）
└── .wiki_state.json        # 内部状态
```

## Compile 反馈闭环设计

```
Promote 拒绝
    │
    ▼
compile_feedback.md （追加拒绝原因）
    │
    ▼
下次 Compile 读取并注入 LLM prompt
    │
    ▼
LLM 参考历史拒绝信号调整编译策略
```

compile_feedback.md 格式：
```markdown
# Compile 反馈记录
# 此文件由 wiki promote --reject 自动追加，供 Compile 阶段参考

## 2026-04-17 — compiled/2026/04/xxx.md
- 原因：内容太碎片化，缺少核心观点
- 建议：合并相关素材后再编译

## 2026-04-18 — compiled/2026/04/yyy.md
- 原因：与已有条目 [[ai-agent]] 重复
- 建议：更新已有条目而非创建新条目
```

## 命令变更对照

| v0.1.0 | v0.2.0 | 说明 |
|--------|--------|------|
| `wiki ingest --text "..."` | `wiki capture --text "..."` | 仅存入 raw/，不编译 |
| `wiki ingest --url "..."` | `wiki capture --url "..."` | 仅抓取存入 raw/ |
| `wiki ingest --file <path>` | `wiki capture --file <path>` | 仅复制到 raw/ |
| `wiki ingest` (扫描 raw/) | `wiki compile` | 编译 raw/ → compiled/ |
| `wiki promote <output>` | `wiki promote [file]` | 支持 compiled/ 审计流 |
| — | `wiki compile --file <path>` | 编译指定 raw 文件 |
| — | `wiki promote` (无参数) | 列出待审核项 |
| — | `wiki promote --reject <f> --reason "..."` | 拒绝并反馈 |
| — | `wiki gc` | 垃圾回收 |
| `wiki ingest` | (保留，打印 deprecation) | 向后兼容 |

## OpenClaw Plugin 部署

Capture Plugin 以 skill 形式部署到 OpenClaw agent：

```
~/openclaw/agents/<agent>/skills/wiki-capture/ → /path/to/personal-wiki/skill/wiki-capture/
```

Plugin 工作方式：
1. OpenClaw 在每轮对话中检查消息是否包含 `#wiki`
2. 命中时，提取当前消息 + 前后 N 条上下文
3. 生成带 frontmatter 的 markdown 文件
4. 写入 `$WIKI_ROOT/raw/notes/`

与现有 wiki-knowledge skill 的关系：
- wiki-knowledge: 查询 + 沉淀（已有）
- wiki-capture: 自动采集（新增）
- 两者可同时部署在同一 agent 上
